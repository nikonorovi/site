Mysql replication

Открываем в iptables порт 3306 на обоих серверах, можно открыть только для конкретного хоста.

iptables -I INPUT -p tcp --dport 3306 -j ACCEPT

По умолчанию уже включены бинлоги, если же нет производим:

На сервере, который будет выступать мастером, необходимо внести правки в my.cnf:

# выбираем ID сервера, произвольное число, лучше начинать с 1
server-id = 1
# путь к бинарному логу
log_bin = /var/log/mysql/mysql-bin.log
# название Вашей базы данных, которая будет реплицироваться
binlog_do_db = newdatabase

Перезапускаем Mysql:

Далее выполняем на мастере:

master# slave IP (hiper ip) 
GRANT replication slave ON *.* TO "repluser" @ "IP SLAVE" IDENTIFIED BY "aeTho0Iecah8"; 
FLUSH PRIVILEGES; 
show master status;

Далее блокируем все таблицы в нашей базе данных:

USE newdatabase; 
FLUSH TABLES WITH READ LOCK; 
show master status;

Теперь необходимо сделать дамп базы данных:

mysqldump -u root -p newdatabase > newdatabase.sql

Разблокируем таблицы в консоли mysql:

 UNLOCK TABLES;

На слейве выполняем

CHANGE MASTER TO MASTER_HOST = "IP MASTER", MASTER_USER = "repluser", MASTER_PASSWORD = "aeTho0Iecah8", MASTER_LOG_FILE = "mariadb-bin.000002", MASTER_LOG_POS = 982, MASTER_SSL=1, MASTER_SSL_CA = '/etc/mysql/newcerts/ca-cert.pem', MASTER_SSL_CERT = '/etc/mysql/newcerts/client-cert.pem', MASTER_SSL_KEY = '/etc/mysql/newcerts/client-key.pem';
FLUSH PRIVILEGES;
SLAVE START;
show slave status \G

Реплика готова.
